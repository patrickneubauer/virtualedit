About

The creation of state of the art systems often involves managing a multitude of different languages instances and their interrelations. To support various types of stakeholders, which have different cross-domain perspectives, distinctive views of the system may be required.

Developing in an aspect oriented way means being oblivious to the complete program. We want to give modelers and programmers back the concern about the program they are interested in.

Here, we present a generic virtual textual model composition approach built on model-driven engineering which allows developers to dynamically select the aspects they are interested in.

We have built a showcase for simplified Java and Aspect, but we support aspect oriented modelling for any languages.

This approach might help debugging and extending your models and programs

Getting Started

You can obtain the latest version of IntellEdit by directly downloading the IntellEdit project archive file or clone the IntellEdit GitHub repository.

Watch IntellEdit in action by proceeding with the following simple steps:

  1. Optional: Download the VirtualBox Image and uncompress it using 7-zip.
  2. Recommended: Clean Eclipse workspace.
  3. Recommended: Right-click on "at.ac.tuwien.big.serviceexample/model/serviceexample.ecore" and register the package.
  4. Recommended: Open the file "at.ac.tuwien.big.servicelang/src/at.ac.tuwien.big.servicelang/Service.xtext", perform a change in the same file, revert the change, and save the null change (this will remove any errors displayed in the grammar editor).
  5. Run an Eclipse runtime application, i.e., a second Eclipse instance started by your initial Eclipse instance, by right-clicking on any imported project in the package explorer and select "Run As > Eclipse Application".
  6. In case the recommended steps have been skipped, an error pop-up may appear in which "Proceed" needs to be selected.
  7. Within the Eclipse runtime application you can start by creating a new project as well a new file fitting our examplary DSML, i.e., having the extension ".service".
  8. Copy the content of the file "at.ac.tuwien.big.serviceexample/model/test.service" to the ".service" file you just created.
  9. Try out advanced editor features !

Note that validation is performed as soon as file changes have been saved.

Example Language Metamodel

This exemplary language metamodel is composed of the classes Cluster, Service, and Server. All classes, are provided with a name attribute.

A Service is defined in terms of designSpeed, i.e., an integer representing the speed for which a service is intended to be used, type, which may either be BESTEFFORT, WEAKCONTRACT, FAILSAFE, or IMPORTANT, and has to be providedBy exactly one cluster.

A Cluster has a designSpeed, i.e., an integer representing the speed for which a cluster is intended to be used, as well and one or multiple containing server. Moreover, a cluster is associated with one or more services and up to one backup cluster.

Finally, each Server has a speed that it provides.

Example Language Constraints

This exemplary language defines formal constraints for clusters, services, and servers.

A Cluster is restricted by sameServices, i.e., its eventually associated backup cluster being associated with all services that are associated by the cluster itself, goodSpeed, i.e., the sum of speeds provided by contained servers has to be equal or greater than the speed for which the cluster itself has been designed for, onlyOneImportant, i.e., only one important service may be defined, and backupDifferent, i.e., the backup cluster can not be the cluster itself.

A Service is constrained by the type of service and its associated speed requirements. In detail, BESTEFFORT services have no further restrictions. Moreover, WEAKCONTRACT services need to have a cluster that has been designed with a speed equal or greater than the designed speed of themselves. Further, FAILSAFE services are further constrained by requiring a backup cluster. Additionally, IMPORTANT services are further constrained by requiring the designed speed of their backup cluster to be at least or greater than the designed speed of the service itself.

The valid speed of a Server is restricted to numbers greater than zero and less or equal five.

Example Model in enhanced Editor

This example model has a single important service with speed 4 that ought to be provided by the WebCl cluster.

However, the WebCl cluster only has a Backup cluster with speed 3. Hence, the  design speed of the backup cluster is lower than the speed required by the main service.

Example Local Search Quick Fix Solution

IntellEdit prioritizes quick fix solutions in the context menu.

Local seach quick fix solutions are marked green and global search quick fix solutions are marked blue.

The first suggestion, i.e., Set .designSpeed to 3, is a local search quick fix solution that reduces the designSpeed as little as possible to make the model valid again.

Example Global Search Quick Fix Solution

IntellEdit prioritizes quick fix solutions in the context menu.

Local seach quick fix solutions are marked green and global search quick fix solutions are marked blue.

A lower prioritized suggestion such as the selected Multiple Changes represents a solution found by the genetic search algorithm that fixes the model through multiple changes.

Quick fix solutions that are generated by the genetic search algorithm can show their full potential in large models, which involve complex model changes that are difficult to establish by human editor users.
